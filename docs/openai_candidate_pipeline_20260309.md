# OpenAI 후보 재심사 / 내러티브 판정 파이프라인

## 목적
- 실시간 매매 엔진 전체를 LLM에 맡기지 않는다.
- 기존 룰베이스/점수 엔진이 추린 `상위 후보`만 LLM이 재심사한다.
- 월 `30 USD` 예산을 넘기지 않도록 `예산 가드`를 먼저 둔다.

## 적용 범위
1. `Candidate Review`
   - 대상: `THEME_SNIPER` 상위 후보
   - 입력: 상위 `TOP N` 요약 JSON
   - 목적: 진입 우선순위 재정렬, 과열/복제/소셜 노이즈 판별

2. `Narrative Review`
   - 대상: `NARRATIVE` 트리거 후보
   - 입력: 재점화 후보 1~3개
   - 목적: 죽은 코인의 내러티브 부활 여부 판별

## 예산 설계
- 모델: `gpt-5-mini`
- 월 예산: `30 USD`
- 일 예산: `0.85 USD`
- 후보 재심사 주기: `600s`
- 후보 수: `TOP 8`
- 추정 입력 토큰: `2600`
- 추정 출력 토큰: `220`
- 내러티브 최대 호출: `12/day`

## 기대 사용 패턴
- Candidate Review:
  - 10분마다 1회
  - 하루 최대 144회 이론치이지만, 일 예산 가드가 먼저 막는다
- Narrative Review:
  - 이벤트성 호출만 허용
  - 하루 12회 상한

## 데이터 흐름
1. 밈 엔진이 `THEME/SNIPER/NARRATIVE` 후보를 점수화
2. 상위 후보만 JSON 요약으로 패킹
3. 예산 가드 확인
4. OpenAI 호출
5. 결과를 아래 용도로 사용
   - 우선순위 재정렬
   - 진입 보류 이유 설명
   - 텔레그램/대시보드 요약

## 현재 코드 상태
- 설정 추가 완료
- 예산 상태 파일 추가 완료
- 대시보드 상태 노출 완료
- 실제 매매 결정에 강제 반영은 아직 미적용
