Step 04

4개 모델이 entry·TP·SL을 직접 제안하는 planner 구조로 바꾼 이유

단순 점수형 모델에서 진입가, 손절가, 목표가를 제안하는 planner 모델 구조로 바꾼 이유와 모델별 역할을 설명합니다.

게시일 2026-03-15 수정일 2026-03-15 키워드 트레이딩 모델, entry, stop loss, target price, planner model, AI_Auto

점수만 높다고 바로 들어가면 생기는 문제

초기 모델은 점수가 일정 기준을 넘으면 현재가 기준으로 진입하는 성격이 강했습니다. 이 구조는 구현은 쉽지만, 왜 그 가격에서 들어가야 하는지와 손절과 목표가가 어떤 논리로 잡혔는지를 설명하기 어렵습니다.

특히 선물 구조에서는 진입 시점과 손절 폭이 레버리지 해석에 직접 영향을 주기 때문에, 점수 하나만으로 포지션을 여는 방식은 운영 화면과도 잘 맞지 않았습니다. 그래서 지금은 각 모델이 단순한 예측 점수기가 아니라 trade planner처럼 행동하게 바꿨습니다.

현재 모델이 내놓는 값

지금의 모델은 score만 뱉지 않습니다. entry_price, entry_zone_low, entry_zone_high, stop_loss_price, target_price_1/2/3, risk_reward, recommended_leverage를 함께 만듭니다. 즉 신호를 내는 것이 아니라 실행 계획을 제안하는 구조입니다.

이 덕분에 `/positions` 화면에서 어떤 가격대가 제안되었고, 체결이 어떤 경로로 반영되었는지를 보여주기 쉬워졌습니다. 모델 성과와 포지션 화면의 정보도 이 planner 출력 덕분에 더 자연스럽게 연결됩니다.

모델별 역할을 분리한 이유

모델 A는 레인지 리버전, 모델 B는 리클레임, 모델 C는 압축 돌파, 모델 D는 리셋 바운스처럼 각자의 성격을 분리했습니다. 이렇게 하면 같은 시점에 서로 다른 상황을 읽는 모델이 공존하더라도, 왜 그런 신호가 나왔는지 설명이 가능합니다.

또한 성과 비교도 단순히 A/B/C/D라는 코드명이 아니라 전략 성향으로 읽을 수 있게 됩니다. 운영자 입장에서는 모델별 PnL보다 먼저, 이 모델이 어떤 성격의 기회를 잡으려는지 이해해야 튜닝 결과도 납득할 수 있습니다.

점수는 여전히 중요하지만 역할이 바뀌었다

점수 자체를 버린 것은 아닙니다. 점수는 진입 비중과 recommended leverage, 현재 setup을 채택할지 말지를 판단하는 입력값으로 남아 있습니다. 다만 점수가 모든 것을 결정하는 것이 아니라, planner가 만든 가격 구조 위에서 점수가 배치 비중을 조절하는 방식으로 역할이 옮겨졌습니다.

planner 구조가 운영 화면과 잘 맞는 이유

planner 모델은 단순히 점수를 내는 구조보다 운영자가 판단하기 쉽습니다. 점수만 보여주면 결국 왜 들어갔는지, 어디서 잘라야 하는지, 목표를 어떻게 잡았는지가 따로 해석돼야 합니다. 하지만 entry, stop loss, target price를 직접 만들면 모델이 어떤 시나리오를 보고 있는지 훨씬 선명해집니다.

이 차이는 데모 검증에서도 큽니다. 계획 가격이 남아 있어야 나중에 intrabar 시뮬레이션으로 실제 터치 여부를 판정할 수 있고, 성과 화면에서도 모델별로 어떤 구간에서 강점이 있었는지 읽기가 쉬워집니다. 즉 planner 구조는 신호 생성과 운영 기록 양쪽에 모두 유리합니다.

모델 이름과 역할을 함께 고정해야 하는 이유

운영 화면에서 모델을 A, B, C, D로만 보여주면 성과를 읽을 때 맥락이 약해집니다. 그래서 레인지 리버전, 리클레임, 압축 돌파, 리셋 바운스처럼 역할 이름을 붙이는 것이 중요합니다. 이름이 붙어 있어야 손익 변화가 단순한 숫자가 아니라 어떤 전략 성격에서 나왔는지 바로 해석할 수 있습니다.
