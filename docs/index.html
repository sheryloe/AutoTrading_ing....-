Model matrix

Four planner models, four execution attitudes

The models are meant to disagree in useful ways. They should create different entry zones, different invalidation logic, and different target pacing.

A Range Reversion

Mean reversion around stretched local ranges. Defensive leverage band and fast invalidation.

  • Entry: support-side reversion zone
  • Stop: range breakdown
  • Targets: mid-band to opposite edge
B Support Reclaim

Recovery trades that wait for price to reclaim structure before taking risk.

  • Entry: reclaim confirmation band
  • Stop: failed reclaim
  • Targets: swing recovery ladder
C Compression Breakout

Expansion trades built from squeezed volatility and retest behavior.

  • Entry: breakout retest
  • Stop: box re-entry
  • Targets: expansion multiples
D Reset Bounce

Washout and rebound setups that prioritize a clean reset before re-entry.

  • Entry: bounce after local reset
  • Stop: reset low failure
  • Targets: rebound levels

FAQ

Operational questions first

Is Vercel supposed to run the Flask engine?

No. The current direction is to split the frontend from the Python runtime. Vercel should host the frontend, not the always-on engine loop.

Do provider keys still belong in GitHub Secrets?

No for service mode. Bybit, Binance, and CoinGecko keys are intended to be saved in the Vercel service console and encrypted into Supabase.

Does saving Bybit keys mean live orders are running?

No. Saving Bybit keys only marks the execution provider as configured. Live intent is a separate target plus arm flow, and the current crypto path is still prepared for future live routing rather than active live order placement.
