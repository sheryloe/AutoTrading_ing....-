한 화면에 다 넣지 않고, 운영 흐름에 맞게 나눈 자동매매 콘솔

AI_Auto는 상시 자동매매 화면을 단순히 예쁘게 포장한 프로젝트가 아닙니다. 개요, 모델 성과, 포지션, 설정을 분리해 운영자가 실제로 읽고 판단하기 쉬운 구조로 다시 설계했습니다.

서비스 콘솔

거래소 키와 runtime profile을 운영 화면에서 분리 관리합니다.

왜 재구성했나

대시보드, 성과, 설정이 섞이면 운영 화면이 아니라 잡탕이 됩니다.

역할 분리

운영 상태 확인, 성과 비교, 포지션 점검, 설정 입력은 같은 순간에 같은 밀도로 보여줄 필요가 없습니다.

판단 속도

지표를 보다가 설정 폼을 만나는 구조는 운영 판단을 느리게 만듭니다. 그래서 화면을 기능별로 분리했습니다.

제품 인상

GitHub Pages는 앱 복제본이 아니라 브랜드 랜딩으로 두고, 실제 콘솔은 Vercel 앱으로 역할을 나눴습니다.

화면 구조

운영자 시선에 맞춘 4개 화면 구조

01 개요

핵심 KPI, heartbeat, 최근 사이클 요약, 오픈 포지션 수만 보여줍니다.

02 모델 성과

모델별 PnL, 승률, 종료 거래 수, autotune 상태를 비교합니다.

03 포지션

오픈 포지션, 최신 setup, entry / SL / TP 상태를 실행 기준으로 모읍니다.

04 설정

Service control, provider vault, execution target, runtime profile은 설정 화면에만 둡니다.

화면 분위기

하이퍼리퀴드 계열의 다크 운영 도구 인상

검은 배경, 민트와 시안 포인트, 데이터 패널 중심 레이아웃을 사용했습니다. 과한 소개 문구보다 운영 도구처럼 보이는 질감을 우선했습니다.

  • 페이지별 책임 분리
  • 데이터 카드와 표 중심 레이아웃
  • 설정 입력은 별도 화면으로 격리

모델 운영

4개 모델이 신호와 계획 가격을 생성합니다.

A 반전형

되돌림 구간과 평균 회귀를 중심으로 진입 가격을 계산합니다.

B 유동성 회수형

스윕 후 회복되는 패턴에서 entry / stop 구조를 만듭니다.

C 압축 돌파형

변동성 수축 이후의 breakout retest를 기준으로 설정합니다.

D 실패 돌파형

실패한 돌파 이후 반대 방향으로 복귀하는 구간을 활용합니다.
