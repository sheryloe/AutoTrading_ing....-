운영 체크

즉시 점검 순서

  1. Heartbeat가 최근 2분 내 갱신되는지 확인
  2. `model_setups`/`model_signal_audit` 최신 cycle 증가 확인
  3. `last_bybit_sync_ts`와 preflight 상태 확인
  4. 오류 시 self-hosted runner 서비스 상태 확인

구축 방법

PowerShell 기준 배포/운영 명령

# runner 설치/등록 (관리자 PowerShell)
.\ops\setup-self-hosted-runner.ps1 -Repo "sheryloe/Automethemoney" -ReplaceExisting

# cloud-cycle 수동 1회 실행
.\ops\run-once-cloud-cycle.ps1 -Repo "sheryloe/Automethemoney" -Workflow "cloud-cycle.yml" -Ref "main"

# Supabase 반영 검증
.\ops\verify-stack.ps1 -EnvFile ".\\.env" -LookbackHours 1

아키텍처

서비스 구성

  • Self-hosted runner: 스케줄 배치 실행 (`cloud-cycle`)
  • Supabase: 상태 원장 (`engine_heartbeat`, `model_setups`, `positions`, `daily_model_pnl`)
  • Vercel: 운영 콘솔/설정 API
  • GitHub Pages: 운영문서 + 상태허브
