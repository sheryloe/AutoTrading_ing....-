Model lineup

A/B/C/D strategy overview

Model A

Trend-following profile. Prioritizes directional continuation after structural confirmation.

Model B

Mean-reversion profile. Focuses on pullback recovery zones and volatility normalization.

Model C

Breakout profile. Uses range expansion and momentum acceleration for entry timing.

Model D

Adaptive profile. Uses runtime tuning with stricter drawdown controls and fast reconfiguration.

Service flow

How the service is delivered

  1. Market data and context are collected from exchange/public sources.
  2. Models A/B/C/D generate setups and signal audit records per cycle.
  3. Paper positions and PnL are updated and written to Supabase.
  4. Vercel console and this hub read the latest state for operators.

Hosted setup

Build and operate (PowerShell)

python -m pip install --upgrade pip
pip install -r .\requirements.txt

# Run one hosted cycle
.\ops\run-once-cloud-cycle.ps1 -Repo "sheryloe/Automethemoney" -Workflow "cloud-cycle.yml" -Ref "main"

# Verify state writes
.\ops\verify-stack.ps1 -EnvFile ".\.env" -LookbackHours 1 -ExpectedRunner "github-hosted"

Required secrets

Minimum configuration

  • GitHub Secrets: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `SERVICE_MASTER_KEY`, `BYBIT_API_KEY`, `BYBIT_API_SECRET`
  • Vercel Env: Supabase URL/keys + `SERVICE_MASTER_KEY` + `SERVICE_ADMIN_TOKEN`
  • Runtime policy: `TRADE_MODE=paper`, live execution flags disabled, readonly Bybit sync enabled
