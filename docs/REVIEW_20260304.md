# AutoTrading 구조 검토 리포트 (2026-03-04)

## 1) 검토 범위
- 무료 API 의존성/추가 필요 여부
- 6시간 자동 튜닝 동작 여부
- 현재 구조 안정성(분리, 초기화 보호, 운영성)

## 2) 결론 요약
- 무료 API는 **추가 필수 없음**. 현재 구조만으로 데모/튜닝 운영 가능.
- 자동 튜닝은 코드/상태 기준으로 **정상 동작 중**.
- 밈/크립토 6런 분리 + 초기화 보호 + 상태 백업이 반영된 안정 구조.

## 3) 무료 API 현황 점검

### 이미 사용하는 무료/공개 소스
- DexScreener 공개 API (토큰/페어 스냅샷)
- Pump.fun 프론트 API (신규/최근 코인)
- CoinGecko (demo key 옵션)
- Yahoo RSS (뉴스)
- Reddit RSS (커뮤니티)
- Binance 공개 시세/캔들
- Bybit 공개 시세
- Solscan Pro API (키 필요, 무료 티어 CU 기반)
- Google Gemini (키 필요, 무료 티어 한도 기반)

### 현재 상태에서 “추가로 꼭 필요한” 무료 API
- 없음

### 있으면 좋은 선택 옵션(필수 아님)
- GDELT/NewsAPI 대체 뉴스 소스(장애시 fallback 강화)
- Fear & Greed, Funding/OI 공개 집계 소스(크립토 리스크 필터 보강)

## 4) 6시간 튜닝 동작 검증

### 코드 레벨 근거
- `src/engine.py`
  - `MODEL_AUTOTUNE_INTERVAL_SECONDS = 21600` (6시간)
  - `_maybe_autotune_models()`에서 `next_eval_ts` 기준 평가/갱신
  - 평가 지표: closed/win_rate/pnl/profit_factor

### 상태 파일 근거 (실행 시점)
- `state.json`의 `crypto_A/B/C -> model_runtime_tune`
- 확인값(검토 시점):
  - A: `thr=0.078, tp=1.00, sl=0.82, note=hold_not_enough_samples`
  - B: `thr=0.052, tp=1.24, sl=1.02, note=hold_not_enough_samples`
  - C: `thr=0.062, tp=1.39, sl=1.12, note=aggr_risk_off`
- `next_eval_ts`가 모두 존재하고 남은 시간 계산 가능 -> 주기 동작 정상

## 5) 현재 구조 안정성 체크
- 6런 분리: `meme_A/B/C`, `crypto_A/B/C` 활성
- 레거시 보존: `legacy_A/B/C` 백업 유지
- 초기화 보호:
  - 기본 `ALLOW_DEMO_RESET=false`
  - reset은 unlock + 확인문구 필요
- 상태 백업:
  - 주기 백업(`reports/state_backups/`) + reset 전 백업

## 6) 권장 운영 파라미터 (무료 한도 보호)
- `GOOGLE_TREND_INTERVAL_SECONDS=1800`
- `GOOGLE_TREND_COOLDOWN_SECONDS=21600`
- `TREND_ERROR_BACKOFF_SECONDS=900`
- `SOLSCAN_BUDGET_WINDOW_SECONDS=300`
- `TELEGRAM_REPORT_INTERVAL_SECONDS=600`
