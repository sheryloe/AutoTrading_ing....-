# Step 3. Top 5 메이저 코인과 선물 유니버스로 범위를 줄인 이유

## 범위를 줄이는 것이 성능보다 먼저였던 이유
자동매매 프로젝트가 흔들리는 가장 큰 이유 중 하나는 유니버스를 너무 넓게 잡는 데 있습니다. 대상 종목이 많아질수록 신호 품질, 데이터 품질, 체결 가정, 설명 가능성이 동시에 나빠지기 쉽습니다. 이번 프로젝트는 이 문제를 피하기 위해 처음부터 Top 5 메이저 코인만 다루는 구조로 고정했습니다.

현재 대상은 BTCUSDT, ETHUSDT, SOLUSDT, XRPUSDT, BNBUSDT입니다. 밈 코인은 제외했고, 시총과 유동성이 높은 선물 심볼만 남겼습니다. 이 선택은 화려함을 줄이는 대신 운영 안정성을 높이는 방향입니다.

## spot이 아니라 futures라는 전제
이번 재구성에서 중요한 것은 포지션 비중과 레버리지를 선물 기준으로 다시 잡았다는 점입니다. 현재 기본 가정은 모델 점수에 따라 진입 비중 10%~30%, 레버리지 5x~25x 범위 안에서만 움직이는 futures demo입니다.

이 말은 기존에 spot처럼 이해되던 체결 기록이나 포지션 해석을 그대로 이어가면 안 된다는 뜻이기도 합니다. 바로 이 때문에 하드 리셋을 통해 과거 누적치를 비우고, 현재 선물 기준의 시드와 규칙으로 새로 시작하는 과정을 별도로 넣었습니다.

## provider를 나눈 이유
Bybit는 실행 계정, Binance는 실시간 시세 소스, CoinGecko는 시총과 메타 데이터 보강 소스로 분리했습니다. 같은 거래소 하나에 모든 역할을 몰아넣는 구조는 단순해 보여도 장애와 품질 문제를 함께 안고 갑니다.

예를 들어 1분봉 intrabar 판정은 Binance 데이터를 우선하고, 실행 키는 Bybit만 사용하게 하면 역할이 분명해집니다. provider를 분리하는 일은 단순한 API 키 관리가 아니라, 어떤 데이터가 어떤 책임을 갖는지 명시하는 설계 작업입니다.

## 메이저 5종만 다룰 때 생기는 이점
대상 심볼이 제한되면 모델별 신호를 비교하기가 쉬워지고, 8분 배치와 1분 intrabar 시뮬레이션도 현실적인 비용 안에서 운영할 수 있습니다. 또한 오탐 하나가 바로 전체 전략을 왜곡하는 문제도 줄어듭니다. 메이저 코인은 밈 코인보다 손절과 목표가 구조를 planner 방식으로 설명하기도 수월합니다.

## 왜 Top 5 futures 데모가 실험 단위로 적절한가
범위를 줄인다는 것은 단순히 코인 개수를 줄인다는 뜻이 아닙니다. 시장 구조가 너무 다른 자산을 억지로 한 시스템에 넣지 않는다는 뜻에 가깝습니다. 밈 코인은 뉴스와 유동성 쏠림에 의해 성격이 급격히 바뀌기 때문에, planner 모델과 intrabar 체결 규칙을 검증하는 단계에서는 오히려 잡음을 크게 만듭니다.

반대로 메이저 5종 선물은 호가와 거래량이 비교적 안정적이고, 8분 배치 기준의 데모 체결 시뮬레이션도 검증하기 좋습니다. 범위를 이렇게 고정하면 모델의 장단점을 더 또렷하게 읽을 수 있고, 시드와 레버리지 정책을 조정할 때도 비교 기준이 흔들리지 않습니다.

## provider 분리가 유니버스 관리에도 주는 효과
CoinGecko는 시가총액과 메타 정보를 정리하는 데 강하고, Binance와 Bybit는 실제 시장 가격과 선물 구조를 읽는 데 강점이 있습니다. 이 세 소스를 역할별로 나누면 어떤 데이터가 유니버스 선정에 쓰였고 어떤 데이터가 실행 판단에 쓰였는지 더 분명해집니다. 범위를 줄이는 작업은 결국 데이터 책임을 선명하게 만드는 작업이기도 합니다.
