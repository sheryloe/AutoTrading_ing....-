# Telegram 409 충돌 조치 기록 (2026-03-04)

## 증상
- 텔레그램 폴링 오류 `409 Conflict`
- 의미: 동일 봇 토큰으로 `getUpdates`가 중복 호출됨

## 원인
- 로컬에서 Python 프로세스가 2개 동작:
  - `web_app.py`
  - `launcher.py`
- 두 프로세스가 같은 텔레그램 봇 토큰으로 polling 수행 가능

## 코드 조치
- `src/engine.py`에 프로세스 간 폴링 락 추가
  - 락 파일: `reports/telegram_poll.lock`
  - stale 기준: 120초
  - 락 점유 중이면 polling skip + 경고 알림
  - 종료 시 락 해제

## 운영 조치
1. 중복 실행 프로세스 종료
2. `web_app.py` 단일 인스턴스 재시작
3. `/status`, `/errors` 확인

## 확인 포인트
- `docs/REVIEW_20260304.md`와 함께 운영
- 409 재발 시 먼저 중복 프로세스 확인:
  - `Get-CimInstance Win32_Process -Filter "name='python.exe'" | Select ProcessId,CommandLine`
